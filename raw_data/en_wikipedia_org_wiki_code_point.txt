From Wikipedia, the free encyclopedia

Numerical value representing a character in a coded character set

Not to be confused with [point code](https://en.wikipedia.org/wiki/Point_code "Point code").

A **code point**, **codepoint** or **code position** is a particular position in a [table](https://en.wikipedia.org/wiki/Table_(database) "Table (database)"), where the position has been assigned a meaning. The table may be one dimensional (a column), two dimensional (like cells in a spreadsheet), three dimensional (sheets in a workbook), etc... in any number of dimensions.

Technically, a code point is a unique position in a quantized n-dimensional space, where the position has been assigned a semantic meaning. The table has discrete (whole) and positive positions (1, 2, 3, 4, but not fractions).

Code points are used in a multitude of formal information processing and telecommunication standards.[[1]](https://en.wikipedia.org/wiki/Code_point#cite_note-:0-1)[[2]](https://en.wikipedia.org/wiki/Code_point#cite_note-2) For example ITU-T Recommendation T.35[[3]](https://en.wikipedia.org/wiki/Code_point#cite_note-3) contains a set of country codes for telecommunications equipment (originally fax machines) which allow equipment to indicate its country of manufacture or operation. In T.35, Argentina is represented by the code point 0x07, Canada by 0x20, Gambia by 0x41, etc.

In character encoding
---------------------

[[edit](https://en.wikipedia.org/w/index.php?title=Code_point&action=edit&section=1 "Edit section: In character encoding")]

Code points are commonly used in [character encoding](https://en.wikipedia.org/wiki/Character_encoding "Character encoding"), where a code point is a numerical value that maps to a specific [character](https://en.wikipedia.org/wiki/Character_(computing) "Character (computing)"). In character encoding code points usually represent a single [grapheme](https://en.wikipedia.org/wiki/Grapheme "Grapheme")—usually a letter, digit, punctuation mark, or whitespace—but sometimes represent symbols, [control characters](https://en.wikipedia.org/wiki/Control_character "Control character"), or formatting.[[4]](https://en.wikipedia.org/wiki/Code_point#cite_note-4) The set of all possible code points within a given encoding/character set make up that encoding's _codespace_.[[5]](https://en.wikipedia.org/wiki/Code_point#cite_note-5)[[6]](https://en.wikipedia.org/wiki/Code_point#cite_note-6)

For example, the character encoding scheme [ASCII](https://en.wikipedia.org/wiki/ASCII "ASCII") comprises 128 code points in the range 0[hex](https://en.wikipedia.org/wiki/Hexadecimal "Hexadecimal") to 7F hex, [Extended ASCII](https://en.wikipedia.org/wiki/Extended_ASCII "Extended ASCII") comprises 256 code points in the range 0 hex to FF hex, and [Unicode](https://en.wikipedia.org/wiki/Unicode "Unicode") comprises 1,114,112 code points in the range 0 hex to 10FFFF hex. The Unicode code space is divided into seventeen [planes](https://en.wikipedia.org/wiki/Plane_(Unicode) "Plane (Unicode)") (the basic multilingual plane, and 16 supplementary planes), each with 65,536 (=2 16) code points. Thus the total size of the Unicode code space is 17×65,536=1,114,112.

In Unicode
----------

[[edit](https://en.wikipedia.org/w/index.php?title=Code_point&action=edit&section=2 "Edit section: In Unicode")]

For Unicode, the particular sequence of bits is called a _[code unit](https://en.wikipedia.org/wiki/Code\_unit "Code unit")_ – for the [UCS-4](https://en.wikipedia.org/wiki/UCS-4 "UCS-4") encoding, any code point is encoded as 4-[byte](https://en.wikipedia.org/wiki/Byte "Byte") ([octet](https://en.wikipedia.org/wiki/Octet_(computing) "Octet (computing)")) [binary numbers](https://en.wikipedia.org/wiki/Binary_number "Binary number"), while in the [UTF-8](https://en.wikipedia.org/wiki/UTF-8 "UTF-8") encoding, different code points are encoded as sequences from one to four bytes long, forming a [self-synchronizing code](https://en.wikipedia.org/wiki/Self-synchronizing_code "Self-synchronizing code"). See [comparison of Unicode encodings](https://en.wikipedia.org/wiki/Comparison_of_Unicode_encodings "Comparison of Unicode encodings") for details. Code points are normally assigned to abstract [characters](https://en.wikipedia.org/wiki/Character_(computing) "Character (computing)"). An _abstract_ character is not a graphical glyph but a unit of textual data. However, code points may also be left reserved for future assignment (most of the Unicode code space is unassigned), or given other designated functions.[_[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation\_needed "Wikipedia:Citation needed")_]

The distinction between a code point and the corresponding abstract character is not pronounced in Unicode but is evident for many other encoding schemes, where numerous [code pages](https://en.wikipedia.org/wiki/Code_page "Code page") may exist for a single code space.[_[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation\_needed "Wikipedia:Citation needed")_]

History
-------

[[edit](https://en.wikipedia.org/w/index.php?title=Code_point&action=edit&section=3 "Edit section: History")]

The concept of a code point dates to the earliest standards for digital information processing and digital telecommunications.

In Unicode, code points are part of Unicode's solution to a difficult conundrum faced by character encoding developers in the 1980s.[[7]](https://en.wikipedia.org/wiki/Code_point#cite_note-7) If they added more bits per character to accommodate larger character sets, that design decision would also constitute an unacceptable waste of then-scarce computing resources for [Latin script](https://en.wikipedia.org/wiki/Latin_script "Latin script") users (who constituted the vast majority of computer users at the time), since those extra bits would always be zeroed out for such users.[[8]](https://en.wikipedia.org/wiki/Code_point#cite_note-8) The code point avoids this problem by breaking the old idea of a direct one-to-one correspondence between characters and particular sequences of bits.
